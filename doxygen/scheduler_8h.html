◆ add_job()

void add_job ( const char * command)

Ajoute une commande à la file des jobs.

Si la file n'est pas pleine, la commande est copiée et ajoutée. Sinon, un message d'erreur est affiché.

Paramètres
commandChaîne de caractères représentant la commande à exécuter.

◆ cleanup()

void cleanup ( )

Libère toutes les ressources et arrête l'ordonnanceur.

Cette fonction met running à 0, signale les threads en attente, et libère la mémoire allouée pour les commandes de la file de jobs.

◆ handle_signal()

void handle_signal ( int sig)

Gère le signal de terminaison (ex: Ctrl+C).

Cette fonction nettoie la file d'attente et arrête le programme proprement.

Paramètres
sigSignal reçu (ex: SIGINT).

◆ print_queue()

void print_queue ( int current_job_index)

Affiche l'état actuel de la file d'attente des jobs.

Cette fonction imprime la liste des jobs en attente, avec une indication du job en cours d'exécution.

Paramètres
current_job_indexIndex du job en cours d'exécution (0 si aucun).

◆ scheduler()

void * scheduler ( void * arg)

Fonction principale du scheduler exécutée dans un thread.

Cette fonction attend qu'un job soit disponible, puis l'exécute en appelant execute_job(). Elle tourne en boucle tant que running est vrai.

Paramètres
argArgument du thread (non utilisé).
Renvoie
Toujours NULL.
